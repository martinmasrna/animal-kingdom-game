- [ ] **Fix `metrics.py` `impact` confound.** `win_rate_when_drawn` uses presence-only draws (`DRAWN_CAVEAT`), so for a deck whose wins and losses run different lengths it biases *every* non-early card's impact independent of card quality (confirmed on cats_midrange: wins avg 14.3 turns vs losses 20.1). **◐ Stopgap shipped (2026-07-08):** the game-length bias is now documented as `IMPACT_LENGTH_CAVEAT` (surfaced in the report caveat + the `per_card_stats` docstring), so nobody reads impact at face value in the meantime. **Real fix still open:** length-normalize the baseline (touches `metrics.py`'s core calculation; re-validate against the test suite + a fresh gauntlet).
- [ ] **Perft-style move-gen regression fixture (from the chess-engine mining).** Chess engines validate move-gen with *perft*: recursively enumerate all reachable positions to a fixed depth, count them, diff against a trusted reference. We have no second engine to diff, but snapshotting legal-action counts over a handful of fixed positions/openings as golden fixtures would catch silent rules/move-gen regressions cheaply (complements the 260-test suite). Rationale in [`../bots/chess-engine-techniques.md`](../bots/chess-engine-techniques.md).
- [ ] **State-representation speed (before NN bots).** Engine uses per-unit `UnitInstance` objects — fine for thousands of greedy games, *not* for MCTS/AlphaZero (millions of cheap clones + batched feature extraction). When there: switch the in-play representation to struct-of-arrays / entity-component (parallel int arrays, array-copy clones) + a tensor per-seat encoding. Localized to `state.py` (the data layer and id-keyed effect registry are already decoupled). Measure clone cost first; keep `UnitInstance` lean (`__slots__`) meanwhile. *Parked until NN bots.*
