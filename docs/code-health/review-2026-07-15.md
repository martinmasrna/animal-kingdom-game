# Systematic code review — 2026-07-15

## Result

Four high-impact integrity risks need attention: checkpointed benchmark samples are not tied to the rules/data that produced them; a legal card action can execute a different hand instance; the config/text invariant is only partly enforced; and synthetic deck setup mutates process-global state. These can silently change game outcomes or balance evidence.

This was a read-only pass. I changed no code or data and did not run tests or simulations, per the active RefereeBot run. The brief reports the suite green at 379.

## Findings

### Critical — benchmark checkpoints can mix rulesets

**Where:** `animal_kingdom/sim/benchmark_set.py:146-160`

`run_key` records pilot, games, seed, field, deck IDs, and only the `--config` path string. It contains neither `cards.json` content nor resolved configuration values (nor map data). A card retune, or an edit to a config file at the same path, can resume and aggregate incompatible matchup samples.

**Blast radius:** The current RefereeBot card-power ruler can emit a plausible but irrecoverably mixed impact table.

**Proposed fix:** Version the identity and fingerprint loaded card/map data plus canonical resolved `Config`; reject old checkpoints. Add tests that change card/config input without changing the path.

**Owner:** Engine / simulation backlog.

### High — card-ID actions lose per-instance legality (Skunk)

**Where:** `animal_kingdom/engine/actions.py:25-47`, `animal_kingdom/engine/effects.py:114-120, 446-480`, `animal_kingdom/bots/determinize.py:57-71`

`legal_placements()` filters locked `UnitInstance`s, but emits `PlaceAction(card_id, target)`. `_take_from_hand()` then reselects among every equal-ID copy and does not enforce the lock. It selects by strength counter, using hand order to break ties.

This confirms the suspected instance-vs-card-ID class behind the Skunk report. A normal bounce appends the locked copy and often masks it. Determinization creates `kept_locked + re-dealt_unlocked`, so equal-counter copies select the locked one even though move generation offered the unlocked one. Restored states with that order have the same defect.

**Blast radius:** TurnBot/RefereeBot worlds can execute an illegal locked copy. Future per-instance hand state has the same risk.

**Proposed fix:** Put `iid` on placement actions, or bind the legal instance in pending placement data and consume that exact one. If the card-ID API must remain, share the exact eligibility/selection predicate between generation and `_take_from_hand`. Add a determinized locked-plus-unlocked duplicate regression.

**Owner:** Engine backlog; this is an execution bug, not a Balance card change.

### High — numeric card effects still bypass config and text guards

**Where:** `animal_kingdom/engine/config.py:1-10`, `animal_kingdom/engine/effects.py:583-630, 1090-1092, 1314-1325, 1512-1543`, `animal_kingdom/tests/test_card_text_consistency.py:1-145`

The invariant says balance constants live in `Config`; the new guard covers food gains, fed thresholds, and food costs. Other printed numeric effects remain literals: Raven draw-three/shuffle-two (`586-587`), Owl look-three (`615`), Impala draw-two (`1091`), Carmilla three sacrifices (`1315`), and several draw-two handlers (`1512`, `1536`, `1541`). Draw-one values are literals too.

**Blast radius:** A text or balance retune can silently diverge from resolution outside the newly guarded food/cost domains.

**Proposed fix:** Classify every printed effect number as either a named Config dial or typed card-intrinsic data, then add semantic text/data guards for draw, strength, delay, and count phrases. Unclassified numeric effects should fail the guard.

**Owner:** Code Health for the policy/guard; resulting values belong to Cards/Balance.

### High — synthetic decks leak into unrelated in-process callers

**Where:** `animal_kingdom/sim/benchmark_set.py:138`, `animal_kingdom/sim/deck_optimizer.py:130-151`, `animal_kingdom/decks.py:38-58`

`register_synthetic()` mutates `PREMADE_DECKS` and `AK_EXTRA_DECKS` with no lifecycle cleanup. `tests/test_benchmark_set.py:20-36` has an autouse restore fixture because otherwise `test_pool` is order-dependent.

**Blast radius:** A later caller can list/schedule a synthetic deck as shipped content; the environment also reaches worker processes.

**Proposed fix:** Pass an explicit registry/deck payload to workers. As an interim, provide scoped registration that snapshots/restores registry and environment in `finally`.

**Owner:** Engine / simulation backlog.

### Medium — public action application trusts callers

**Where:** `animal_kingdom/engine/rules.py:62-79`, `animal_kingdom/engine/effects.py:69-80, 99-111`

`apply_action()` never checks membership in `legal_actions(state)`: arbitrary placements and pending choices enter resolution, while unavailable cards fail later with an incidental `max()` error. `RecorderSession` compensates at `recording/session.py:199`, but the documented engine action contract and future transport callers do not.

**Blast radius:** Trusted bots normally avoid this, but a CLI, replay, remote client, or stale action can bypass connection, covering, cost, and option checks.

**Proposed fix:** Validate at the public boundary and raise stable `EngineError`; reserve a clearly named trusted path only if profiling requires it. Add illegal-placement and stale-choice tests.

**Owner:** Engine backlog.

### Medium — Landmark/type machinery is dormant

**Where:** `animal_kingdom/engine/cards.py:20-25, 76-78`; `animal_kingdom/engine/effects.py:108, 479, 672, 845, 1189, 1248, 1355`

After the animals-only cut, `LANDMARK_IDS` is empty, `is_unit` is always true, and those branches are dead. `strength_min` has no production caller; only its unit test remains (`tests/test_effects.py:659-662`).

**Risk / proposal:** Immediate risk is low, but coverage/comments imply support that data cannot exercise. Decide explicitly between removing it in a focused diff or retaining it behind documented fixtures and contract tests. This is already correctly filed in Engine and is no worse than described.

### Low — loading I/O sits in the package labelled pure engine

**Where:** `animal_kingdom/engine/resources.py:10-19`, `animal_kingdom/engine/config.py:157-172`

Rules/effects are cleanly transport-free, but package-data and config loaders open files inside `engine/`. This is startup I/O, not game-loop I/O, yet contradicts the literal invariant.

**Proposed fix:** Document “pure after construction” or move file adapters outward. Keep state/rules/effects free of I/O.

**Owner:** Code Health.

### Medium — tests are deep in mechanics but uneven at executable boundaries

**Where:** `animal_kingdom/tests/`; `animal_kingdom/cli.py:1-420`

There is strong direct coverage for effects, fuzz/replay, renderer, TUI, recording, bots, and major report/ratings workflows. There is no direct test module importing `cli.py`, and no direct module for several one-off analysis executables (`conquest.py`, `human_scorer.py`, `measure_deck.py`, `metagame_search.py`, `roster_experiment.py`). The action-boundary and Skunk/determinization combination are also absent. This does not claim zero indirect coverage.

**Proposed fix:** Add parser/validation/provenance smoke tests, prioritizing action validation and checkpoint identity over broad line coverage.

**Owner:** Code Health test strategy; individual tests belong with their subsystems.

## Healthy or bounded seams verified

- Production bots contain no premade deck slug/archetype policy. They use generic tags, keywords, geometry, and state features.
- `StateView` strips opponent identities/order (`state.py:134-182, 371-396`). Greedy's belief helper is partition-invariant (`test_greedy_bot.py:218-223`); TurnBot and RefereeBot have no-hidden-info regressions.
- `_find_unit` still means any stack depth, but remaining callers are appropriate: Egg/Grizzly timers are gated top-of-stack before firing (`effects.py:511-534`); queued removal/bounce target an intended instance; reactive source checks only need on-board existence. No additional top-vs-buried bug was found.
- Map/card validators form clear trust boundaries; order-sensitive engine work is sorted for replay determinism; `runner.run_pairs()` balances mover seats and preserves order under workers.
- Card-intrinsic `base_strength` and `food_cost` are correctly in data, and food-cost guard coverage is bidirectional. The configuration finding is limited to other numeric effects.
- Renderer/transport separation is clean: `render/text.py` returns markup without Rich; CLI owns terminal I/O; TUI owns Textual.
